# 3.3.5 Diagramas de secuencia (flujos clave)

> ⬜ **Pendiente de redacción.** **Cubre:** IEEE 1016 (vista dinámica/interacción).
> **Guía:** [2-diseno-ieee-1016.md](../2-diseno-ieee-1016.md).

## Qué debe contener
- Diagramas de secuencia de los flujos críticos: login con JWT/refresh, alta de tenant, marcar
  asistencia con verificación gRPC CourseManagement, generación/cobro de deuda con Todotix y el
  patrón Outbox → RabbitMQ → consumidor idempotente.

## Diagramas
- ⚠️ Doxygen/Compodoc **no autogeneran** diagramas de secuencia. Definir su fuente en la ola de
  esta sección. No redactar UML embebido hasta acordar la fuente.

## Comandos de demostración (sugeridos)
```bash
grep -rln "Outbox\|Inbox" apps/*/Backend --include=*.cs        # flujo de mensajería
grep -rln "Grpc\|gRPC" apps/Attendance/Backend --include=*.cs  # borde Attendance → CourseManagement
```
