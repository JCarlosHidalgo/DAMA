# 3.2.2 Roles de usuario (multitenancy)

> ⬜ **Pendiente de redacción.** **Cubre:** ISO/IEC/IEEE 29148 (características de los usuarios).
> **Guía:** [1-requisitos-iso-29148.md](../1-requisitos-iso-29148.md).

## Qué debe contener
- Los roles reales: **Admin** (operador global de tenants), **Client** (gestor de su academia),
  **Teacher**, **Student**.
- El modelo de multitenancy (aislamiento por `TenantId`) y cómo cada rol opera dentro de su tenant
  (solo Admin cruza tenants).

## Comandos de demostración (sugeridos)
```bash
grep -rn "UserRoles" apps/*/Backend --include=*.cs | head
grep -rln "IClaimContext" apps/*/Backend --include=*.cs
```
