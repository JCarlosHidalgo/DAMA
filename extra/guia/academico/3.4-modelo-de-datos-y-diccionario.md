# 3.3.4 Modelo de datos y diccionario de datos

> ⬜ **Pendiente de redacción.** **Cubre:** IEEE 1016 (vista de datos).
> **Guía:** [2-diseno-ieee-1016.md](../2-diseno-ieee-1016.md).

## Qué debe contener
- Modelo Entidad-Relación por servicio y diccionario de datos (campo, tipo, nulo, clave,
  descripción), con la columna `TenantId` documentada como mecanismo de aislamiento.
- Justificación de la estrategia de tenancy: esquema compartido con `TenantId`.

## Fuente de verdad
- Los esquemas y *stored procedures* viven en `infrastructure/environments/<svc>/init.sql`.

## Comandos de demostración (sugeridos)
```bash
ls infrastructure/environments/*/init.sql
grep -in "CREATE TABLE\|TenantId" infrastructure/environments/auth/init.sql | head
```
