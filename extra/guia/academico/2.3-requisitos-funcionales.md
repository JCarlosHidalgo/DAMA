# 3.2.3 Requisitos funcionales

> ⬜ **Pendiente de redacción.** **Cubre:** ISO/IEC/IEEE 29148 (RF).
> **Guía:** [1-requisitos-iso-29148.md](../1-requisitos-iso-29148.md).

## Qué debe contener
- Tabla de requisitos funcionales (ID, Tipo, Descripción atómica y verificable, Prioridad, Origen,
  Objetivo asociado), agrupados por dominio (Auth, CourseManagement, Attendance, Payment).
- Formato *"El sistema deberá…"*. Trazables a los endpoints reales.

## Comandos de demostración (sugeridos)
```bash
grep -rn "\[Http" apps/*/Backend --include=*Controller.cs   # endpoints por servicio
ls api-endpoints/                                           # colecciones Bruno por servicio
grep -rn "location /api/" infrastructure                    # rutas expuestas en el gateway
```
